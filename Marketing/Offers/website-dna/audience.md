# Website DNA™ — Audience

## Who This Is For

**AI web developers and vibe coders** — They can spin up a site in an afternoon, but the result doesn't land with clients. They know the tools; they don't know the business. This book fills the gap between "I can build it fast" and "I can build it right for this specific client." They need this because their speed advantage is being undercut by generic output — the client gets something fast but not something theirs.

**Freelancers building client sites** — They compete on price and speed against other freelancers using the same AI tools. When everyone's output looks the same, the cheapest wins. This book gives them a differentiator: the ability to ask the right questions before building, producing work that clients recognize as "theirs." They need this because without it, they're stuck in a race to the bottom on pricing.

**No-code builders (Framer, Webflow, Wix Studio)** — The platform handles the technical side, but they still stare at a blank page or default to templates. They need a system for gathering what makes this business different so their builds feel intentional, not templated. They need this because the tool doesn't solve the strategy problem — it only solves the building problem.

**Small agencies (2–5 people)** — They've adopted AI tools to stay competitive but their junior devs produce work that needs heavy revision. This book gives them a repeatable process they can hand to anyone on the team. They need this because inconsistency kills client trust, and "just prompt better" isn't a process you can delegate.

**Students learning AI-assisted development** — They're learning the tools but not the thinking. This book teaches the discovery and strategy layer that bootcamps and tutorials skip. They need this because "I know how to use Cursor" isn't a portfolio differentiator — everyone knows how to use Cursor.

## Who This Is Explicitly NOT For

**Backend engineers** — This book is about front-end website representation and business understanding. If you're building APIs and databases, the "website that feels like the business" problem isn't your problem. You'd be paying for content that doesn't apply to your daily work.

**Pure graphic designers** — If you already have a deep brand strategy practice and are looking for AI design techniques, this isn't that. This is about understanding a business so the design direction emerges from real context, not about design tool tips or Figma workflows.

**People wanting a prompt list with no strategy behind it** — This book contains a prompt library, but it's built on top of a 7-layer discovery system. If you want to skip straight to "give me 50 prompts for better websites," you'll be frustrated by the first five chapters. The prompts only work because of the context-gathering that comes first — without that, they're just better-worded guesses.
